# תזמון Fetch יומי

## מה אמור לרוץ אוטומטית (ברשימת Actions / Pages)

מכוון מוצר — **שלושה זרמים בלבד** שאינם “רעש” CI:

| מה | איפה מופיע | מתי |
|----|------------|-----|
| **Daily Workout Fetch** | GitHub Actions, workflow `🦆 Daily Workout Fetch` | שלוש ריצות **cron** ביום (להלן) + לעיתים **push** ל־`main` כשמשנים רק `backend/**` או קובץ ה-workflow הזה + **Run workflow** ידני |
| **pages build and deployment** | Actions (workflow מובנה של GitHub Pages) | אחרי **כל push ל־`main`** — כולל אחרי שה־Fetch דוחף `data/workouts.json` / `special_cache.json`, כדי שהאתר ב־Pages יתעדכן |
| **דוח משתמשים (שבועי)** | GitHub Actions, workflow `📊 Weekly Analytics Report` | **פעם בשבוע** — שישי, לפי ה־cron בקובץ ה-workflow (≈07:00 ישראל בקיץ, ≈06:00 בחורף מ־04:00 UTC) + אופציונלית הרצה ידנית |

**שרשרת נכונה:** ה־cron של ה־Fetch רץ → אם יש שינוי בנתונים → commit + push ל־`main` → **Pages** (ו־Vercel אם מחובר ל־Git) נבנים מחדש. בלי push אין עדכון קבצים באתר הציבורי.

Workflows אחרים (Capacitor, בדיקות אנליטיקס ב־CI) מוגדרים כרגע **ידניים** או PR בלבד — לא אמורים להציף את הרשימה על כל push.

## בדיקה: 05:00 / 05:30 ישראל מול 00:01 בכל מקור

**05:30 ישראל** = 03:30 UTC (חורף, UTC+2) או 02:30 UTC (קיץ, UTC+3).  
ב־workflow מוגדר **03:30 UTC** → בחורף בדיוק 05:30 ישראל; בקיץ 06:30 ישראל.

| מקור | איזור | השעה אצלם כש־05:30 בישראל | אחרי 00:01? |
|------|--------|---------------------------|-------------|
| **MYLEO** | גרמניה (CET/CEST) | 04:30 | ✅ כן |
| **Ton Bridge** | UK (GMT/BST) | 03:30 | ✅ כן |
| **1013** | אילינוי, ארה"ב (Central) | 21:30 **אתמול** | ❌ לא – עדיין ערב אתמול אצלם |
| **Restoration** | ארה"ב | 22:30 / 21:30 אתמול | ❌ לא |
| **Official site** | ארה"ב | ערב אתמול | ❌ לא |

**מסקנה:** ב־05:30 ישראל **רק מקורות אירופה/UK** (MYLEO, Ton Bridge) כבר אחרי 00:01. מקורות US יתווספו **בריצה השנייה** (08:00–09:00 ישראל) אחרי שהם מפרסמים בערב אצלם.

## תזמון נוכחי

- **ריצה 0:** 03:00 UTC = **05:00 ישראל** (חורף, UTC+2) — ריצה מוקדמת נוספת; בקיץ (~UTC+3) בערך **06:00** ישראל.
- **ריצה 1:** 03:30 UTC = **05:30 ישראל** (חורף) – אירופה/UK (MYLEO, Ton Bridge).
- **ריצה 2:** 06:00 UTC = **08:00 ישראל** (חורף) – מקורות US (1013, Official site, Restoration).
- **Pages build and deployment:** workflow מובנה של GitHub — רץ אחרי כל push ל־`main` (בדרך כלל כדקה־שתיים אחרי ש־Fetch מסיים ודוחף, אם היה שינוי בקבצים).
