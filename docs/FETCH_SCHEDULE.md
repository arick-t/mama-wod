# תהליך יומי: גריפת אימונים והצגה באפליקציה

מסמך זה מתאר **בסדר כרונולוגי** מה קורה כל יום בין מקורות האימונים לבין המסך אצל המשתמש.  
מקור אמת ל־**קרונים בפועל**: `.github/workflows/daily-fetch.yml`. קוד האיסוף: `backend/fetch_all.py`.

---

## 1. עקרון המפתח: תאריך אחיד = ישראל

| שכבה | איך נקבע "היום" |
|------|------------------|
| **שרת (Python)** | `today_israel()` — `Asia/Jerusalem` (`zoneinfo`) |
| **אפליקציה (דפדפן)** | `todayIsraelStr()` — `toLocaleDateString("en-CA", { timeZone: "Asia/Jerusalem" })` |
| **מפתח ב־JSON** | `workouts["YYYY-MM-DD"]` — אותו פורמט לכל הימים |

כל עוד שני הצדדים משתמשים באותה אזור זמן, **אותו לוח יום** נשמר גם בגריפה וגם בתצוגה.

---

## 2. התהליך היומי — רצף מלא (מא׳ לת׳)

### א. לפני חצות (יום קלנדרי ישראלי D)

- ב־`data/workouts.json` כבר קיימים אימונים לתאריכים האחרונים (עד 14 ימים אחורה אחרי prune).
- משתמש שפותח את האפליקציה רואה את **D** ב"היום" + כרטיסים לפי `workouts[D]`.

### ב. חצות בישראל → מתחיל יום D+1

1. **בממשק:** מתוך רגע אחד, ה"היום" במסך Browse משתנה ל־**D+1** (לפי השעון בדפדפן עם אזור ישראל).
2. **בקבצים הציבוריים:** עדיין ייתכן שאין מפתח `D+1` ב־`workouts.json` עד שהריצה הראשונה של אותו לילה/בוקר תסיים — **זו התנהגות צפויה**, לא בהכרח תקלה.

### ג. GitHub Actions — ריצות אוטומטיות (ביום השעון ב־UTC)

הזמנים הבאים הם **קרובים** ל־**חורף ישראל (UTC+2)**; בקיץ (UTC+3) השעה המקומית בישראל זזה בערך שעה קדימה. ראה טבלה בסעיף 4.

**סדר לוגי ביום קלנדרי חדש:**

| סדר | Cron (UTC) | כוונה (ישראל חורף, בערך) | תפקיד |
|-----|------------|---------------------------|--------|
| 1 | `0 23 * * *` | ~**01:00** | מילוי מוקדם ליום החדש — מצמצם חלון ריק אחרי חצות. |
| 2 | `0 1 * * *` | ~**03:00** | חיזוק מקורות EU/UK. |
| 3 | `0 3 * * *` | ~**05:00** | ריצה מוקדמת נוספת. |
| 4 | `30 3 * * *` | ~**05:30** | מותאם ל־MYLEO / Ton Bridge אחרי 00:01 מקומי אצלם. |
| 5 | `0 6 * * *` | ~**08:00** | מקורות ארה״ב (1013, Restoration, אתר רשמי) אחרי פרסום לילי אצלם. |

בין ריצה לריצה: `fetch_all.py` מתבצעת על הרunner, מעדכנת `workouts.json` + `special_cache.json`, ואז צעד ה־git (**stash → pull --rebase → stash pop → commit → push**) דוחף ל־`main` **רק אם** יש שינוי.

**טריגרים נוספים (לא cron):**

- **Push** ל־`main` שמשנה `backend/**` או את קובץ ה־workflow — מריץ fetch (ימנע רגרסיה בלי לחכות ללילה).
- **Run workflow** ידני מממשק Actions.

### ד. אחרי Push ל־`main`

1. **GitHub Pages** (אם מופעל): בילד מובנה → קבצי `data/*.json` המעודכנים זמינים באתר ה־Pages.
2. **Vercel** (אם מחובר לריפו): דיפלוי לפי `main` — ב־`vercel.json` מוגדר `Cache-Control: no-store` ל־`/data/workouts.json` ול־`/data/special_cache.json` (מקביל ל־`_headers` ב־Pages).
3. **זמן טיפוסי:** בערך **דקה–שתיים** מה־push עד שהאתר מגיש קובץ חדש.

### ה. טעינה והצגה אצל המשתמש (אפליקציית הדפדפן)

1. בטעינת העמוד / בלחיצה **Refresh data**: `loadData()` עושה `fetch("./data/workouts.json?t=" + Date.now())` — **מניעת מטמון אגרסיבית** מהדפדפן.
2. `displayWorkouts()` קורא את `allData.workouts[selectedDateStr]`, מסנן לפי מקורות מופעלים ב־`SOURCES`, וממיין.
3. **פס הימים:** יום מסומן כ"יש בו נתונים" רק אם יש **לפחות אימון אחד** (`length > 0`) לאותו מפתח תאריך.
4. אם **היום** (ישראל) עדיין ריק אחרי טעינה: הודעה מסבירה + כפתור ריענון; ייתכן **ריענון אוטומטי יחיד** ~75 שניות אחרי הטעינה הראשונה (פעם אחת לכל טעינת עמוד).

## 3. למה לפעמים חסרים מקורות בבוקר?

ב־**~05:30 ישראל (חורף)** בדרך כלל כבר מעודכנים מקורות **אירופה/UK** (למשל MYLEO, Ton Bridge) — היום כבר התקדם אצלם אחרי חצות.

ב־**~08:00 ישראל (חורף)** מתאימים לריצה שמכוונת למקורות **ארה״ב** — שם האתרים לעיתים מפרסמים את ה־WOD של "היום" רק בערב המקומי של **אתמול** ביחס לישראל.

(טבלת מקורות מול 00:01 — עדיין רלוונטית; מופיעה בסעיף 5.)

## 4. טבלת UTC ↔ ישראל (לעומק — חורף UTC+2)

| Cron | UTC | ישראל חורף (≈) | הערה |
|------|-----|----------------|------|
| `0 23` | 23:00 | **בוקר למחרת** לוח ישראל (≈01:00) | עוזר מיד אחרי החלפת תאריך מקומי |
| `0 1` | 01:00 | ≈03:00 | |
| `0 3` | 03:00 | ≈05:00 | |
| `30 3` | 03:30 | ≈05:30 | MYLEO, Ton Bridge |
| `0 6` | 06:00 | ≈08:00 | US |

בקיץ (UTC+3) — להזיז את עמודת ישראל בערך **שעה קדימה** (הקרונים נשארים ב־UTC).

## 7. כשהמסך עדיין ריק ל"היום" — צ'קליסט

1. לחכות מעט אחרי חצות / אחרי שעת הקרון הרלוונטית (~01:00 / ~03:00 / ~05:00 / ~08:00 ישראל חורף).
2. **Refresh data** באפליקציה (או רענון מלא לדף).
3. לבדוק ב־[GitHub Actions — Daily Workout Fetch](https://github.com/arick-t/mama-wod/actions/workflows/daily-fetch.yml) האם הריצה האחרונה **success** והאם יש **push** חדש ל־`main` עם `data/workouts.json`.
4. לוודא שהאתר שבו גולשים (Pages / Vercel) כבר סיים דיפלוי אחרי אותו push.
